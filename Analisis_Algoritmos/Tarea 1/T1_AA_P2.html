Pregunta 1.1

Para qu valores de n el algoritmo de insercin (\(8n^2\)) corre ms rpido que el de mezcla directa (\(64nlog_2(n)\))?

Inser1  <- function(n){8*n^2}
Mezcla1 <- function(n){64*n*log2(n)}
n       <- 0:100

ggplot()+ theme_bw()+
  geom_line(aes(n, Inser1(n), col="Insercin"))+
  geom_line(aes(n, Mezcla1(n), col="Mezcla"))+
  xlab("n")+
  ylab("tiempo")+
  ggtitle("Comparacin de los mtodos de ordenamiento de insercin y mezcla")

Pregunta 2

Cul es el valor ms pequeo de n tal que un algoritmo cuyo tiempo de ejecucines \(100n^2\) corre ms rpido que un algoritmo cuyo tiempo de ejecucin es \(2^n\)?

cuad <- function(n){100*n^2}
expo <- function(n){2^n}

n <- 0:20

ggplot()+ theme_bw()+
  geom_line(aes(n, cuad(n), col="Cuadrtico"))+
  geom_line(aes(n, expo(n), col="Exponencial"))+
  xlab("n")+
  ylab("tiempo")+
  ggtitle("Comparacinn de algoritmos")

j <- 1

while(j<20){
  if(expo(j)-cuad(j)>0){
    break
  }
  j <- j+1
}

print(paste0("El valor ms pequeo de n para el que el algoritmo de tiempo cuadrtico 100n^2 corre ms rpido que el exponencial 2^n es: ", round((j),1), "."))
## [1] "El valor ms pequeo de n para el que el algoritmo de tiempo cuadrtico 100n^2 corre ms rpido que el exponencial 2^n es: 15."
